Frequently Asked Questions (FAQ)

Q: How do I activate my license key?

A: In DirectoryCompare Pro, go to Help ‚Üí Register, paste your license key, and click Activate. Your key format should be:

Tier:MachineID:Signature

Example: Premium:CPU123...:SGVsbG8...

Q: Can I transfer my license to a new computer?

A: License keys are hardware-bound to your specific computer. If you upgrade your hardware or switch computers, contact us at dircomparepro@gmail.com with your original license key and new Machine ID. We'll issue a new key for your new hardware.

Q: How do I find my Machine ID for license support?

A: Go to Help ‚Üí Register in DirectoryCompare Pro. Your Machine ID is displayed at the bottom of the registration dialog. It looks like:

CPU:BFEBFBFF00090675_DISK:12345678_MACHINE:DESKTOP-ABC123

Q: What archive formats are supported?

A: DirectoryCompare Pro v1.01 supports:

  • Archives: ZIP, TAR, 7Z, RAR, ISO, CAB, AR, CPIO
  • Compression: GZ, BZ2, XZ, LZMA, LZ4, ZSTD
  • Password protection: ZIP, RAR, 7Z (password dialog appears when needed)

Q: How do I compare GitHub repositories without cloning?

A:

  1. Click "Switch to N-way Mode"
  2. Click "Add Source" ‚Üí Select "GitHub"
  3. Enter GitHub URL: https://github.com/owner/repo
  4. Click "Compare"

Note: For private repos or to avoid rate limits (60 req/hour), set your GitHub token in Settings ‚Üí GitHub Token. This increases your limit to 5,000 req/hour.

Q: How do I create a GitHub personal access token?

A:

  1. Go to GitHub Settings ‚Üí Developer settings ‚Üí Personal access tokens
  2. Click "Generate new token (classic)"
  3. Select scope: repo (for private repos) or public_repo (for public only)
  4. Copy the token and paste it in DirectoryCompare Pro: Settings ‚Üí GitHub Token

Q: Can I compare FTP servers?

A: Yes! DirectoryCompare Pro v1.01 supports FTP, FTPS (TLS/SSL), and SFTP (SSH):

  1. Switch to N-way mode
  2. Add Source ‚Üí Select "FTP"
  3. Click "Configure" to enter server details (host, port, username, password)
  4. Select protocol: FTP (port 21), FTPS (port 990), or SFTP (port 22)
  5. Click "Compare"

Q: How do I find duplicate files using similarity search?

A: Go to Tools ‚Üí Similarity Search:

  • Text files: Use Jaccard or Line Similarity matcher
  • Images: Use dHash or aHash (finds duplicates even if resized)
  • Exact duplicates: Use SHA-256 Content Hash

Set similarity threshold (e.g., 80%) and click "Search". Results show all file pairs above the threshold.

Q: The app shows "License validation failed" - what do I do?

A: Common causes:

  • Wrong Machine ID: License was generated for different hardware
  • Tampered key: Key was modified or corrupted
  • Hardware change: CPU, disk, or machine name changed

Solution: Email dircomparepro@gmail.com with your license key and current Machine ID. We'll issue a corrected key.

Q: How do I reset my trial?

A: Trials are hardware-bound and cannot be reset by deleting files. If your 7-day trial has expired, you can:

  • Purchase a license (Standard $39, Premium $79, Lifetime $199)
  • Submit useful bug reports via in-app dialog to earn free upgrade
  • Use Basic (free) tier with limited features

Q: Does DirectoryCompare Pro work offline?

A: Yes! License validation is fully offline after initial activation. Internet is only required for:

  • GitHub repository comparison (uses GitHub API)
  • FTP/FTPS/SFTP comparison (connects to your servers)
  • Bug report submission (optional - falls back to clipboard if offline)

Q: Is there a Linux version?

A: Yes! DirectoryCompare Pro v1.01 includes full Linux support with 100% feature parity. Download the Linux package from the Download page. Tested on Ubuntu 22.04+, Debian, and Kali Linux. All features work identically on both Windows and Linux, including N-way comparison, Git/GitHub integration, archive support, and enterprise licensing.

Known Issues

Large Directories (100,000+ files)

Issue: Scanning may take several minutes

Workaround: Use file filtering to reduce scope (e.g., *.cpp or src/**)

GitHub API Rate Limits

Issue: 60 requests/hour without token, 5,000 with token

Solution: Set GitHub token in Settings dialog (see FAQ above)

Memory Usage on Large Comparisons

Issue: May use 500+ MB RAM for very large comparisons

Expected behavior: Memory is released after comparison completes. If memory stays high, restart the app.

Troubleshooting

App Won't Start

  • Ensure you're running Windows 10/11 (64-bit)
  • Check if antivirus is blocking the executable (add exception)
  • Download fresh copy from official download page
  • Verify SHA-256 checksum to ensure file isn't corrupted

Comparison Takes Forever

  • Enable "Show Progress" to see what's happening
  • Disable "Deep Compare" (SHA-256 hashing) for faster scans
  • Use file filters to exclude unnecessary directories (e.g., node_modules/, .git/)
  • Click "Cancel" if comparison is stuck

GitHub Comparison Fails

  • Check repository URL format: https://github.com/owner/repo
  • For private repos, set GitHub token in Settings
  • Verify internet connection
  • Check if you've hit rate limit (60 req/hour) - wait or add token

FTP Connection Fails

  • Verify host, port, username, password are correct
  • Check firewall isn't blocking outbound FTP (port 21) or SFTP (port 22)
  • Try passive mode if active mode fails
  • For FTPS, ensure server supports TLS/SSL

Archive Won't Open

  • Check if archive is password-protected (password dialog should appear)
  • Verify archive isn't corrupted (try opening in 7-Zip or WinRAR)
  • For unsupported formats, extract manually first then compare directory
